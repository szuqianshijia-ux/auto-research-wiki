# /wiki-summary

对话结束时生成会话总结，并同步到知识库。

## 工作流

1. **回顾本次对话**的关键发现、决策和产出文件

2. **写会话总结**到 `notes/chat_summaries/YYYYMMDD_类别_主题.md`

   类别参考：训练 / 数据 / 部署 / 调试 / 实验 / 配置 / 分析

   必填结构：
   - 元数据（日期、主题、证据包路径、关联文档、关键词）
   - 这次会话解决了什么
   - 关键结论（1-3 条）
   - 产出文件列表
   - 对项目的影响
   - 后续建议

3. **深度技术细节**放 `docs/YYYYMMDD_主题详情.md`，摘要只链接

4. **同步入库**
   ```bash
   node scripts/sync_notes_to_wiki.mjs --dry-run    # 先预览
   node scripts/sync_notes_to_wiki.mjs --rescan      # 确认后正式同步
   ```

5. **提醒用户**在 LLM Wiki App 中 Rescan Raw Sources

## 判断标准

以下情况**值得写总结**：
- 解决了实际问题（bug fix、配置调优、数据处理）
- 产生了可复用的结论或决策
- 发现了重要的坑或失败原因
- 改变了项目方向或技术路线

以下情况**不需要写总结**：
- 纯闲聊或概念澄清
- 未完成的探索性尝试（无结论）
- 单步操作（文件查看、git 操作）
